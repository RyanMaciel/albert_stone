Methodology

The most important part of this project is the pipeline by which images and metadata have been extracted from RMSC's digital catalog and transformed into usable forms. The first step in creating this pipeline was programming a "web scraper." This is just a program that navigates a website using typical user interactions, such as clicks and scrolls, and saves the data it finds. I developed a scraper that browses the RMSC Albert Stone Catalog and saves the images and their metadata.

After all the data had been retrieved, I began to build what I am calling a "loader." This is another piece of software that takes the saved data and allows the user to annotate it with geolocation data. The program shows the user an image entry that includes the image itself and the relevant metadata. Based on this, the user can tag an image with many location names, addresses, and corresponding latitude/longitude coordinates. The loader then saves this data in a JSON format, which makes it easy to work with using web technologies.

The last piece of the puzzle is the final user interface. The main aspect of which is the interactive, searchable, and filterable map that can be found on the home page of this website. This interactive element uses an open-source mapping software, Leaflet with mapping tiles from MapBox. Its interface takes inspiration from location-based search maps innovated by tech companies such as Airbnb and Zillow. Splitting the presentation of this rich database into the more free-form, spatially oriented interactive map and more formal essay-based explorations gives more flexibility as to how the data can be used.
